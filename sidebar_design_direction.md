# Direção visual aplicada: sidebar esquerdo DOUTORELO

O novo sidebar deve funcionar como navegação principal de produto, não como enfeite lateral. A referência enviada por Sidney aponta para um painel escuro, denso e legível, com item ativo claro, ícones alinhados e hierarquia de produto. A pesquisa reforça que a navegação vertical deve estar à esquerda, ser visível no desktop, trazer rótulos textuais claros e reservar o rodapé para suporte e identidade do usuário.

## Tradução para UI

| Dimensão | Decisão aplicada |
| --- | --- |
| Posição | Sempre à esquerda no desktop; drawer mobile também abre da esquerda. |
| Presença | Sidebar visível, com largura real, sem texto vertical e sem cápsula decorativa. |
| Visual | Azul-marinho profundo com degradê vertical, borda translúcida e brilho sutil no topo. |
| Item ativo | Superfície clara com texto navy, ícone destacado e pequena barra lateral/acento. |
| Hover | Realce com superfície azul translúcida, deslocamento horizontal discreto, ícone mais claro e seta com movimento. |
| Organização | Header de marca, atalho principal, grupos sanfonados, ajuda e perfil no rodapé. |
| Mobile | Botão compacto à esquerda e painel que desliza da esquerda, mantendo o mesmo vocabulário visual. |
| Tom | Chique, moderno, software premium; sem linguagem de bastidor, demo, reunião ou status interno. |

## Critérios de aceitação

A entrega será considerada correta apenas se o componente tiver sidebar esquerdo permanente no desktop, estados hover perceptíveis, item ativo muito claro, organização em grupos de produto, CTA de ajuda no rodapé, perfil no rodapé e testes impedindo regressão para posicionamento à direita ou aparência de botão simples.
